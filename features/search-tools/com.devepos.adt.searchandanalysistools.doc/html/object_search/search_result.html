Working with the Search Results

Once a query is finished the results are displayed in the Search View. For each found object there are several options available

  • Opening the object in the corresponding eclipse editor (if one is registered for the type) by double clicking on it
  • Open one or several selected objects via the context menu item Open
  • Assign Tags to the selected Objects
    The ABAP Tags plug-in needs to be installed and the ABAP Tags backend must be available in the selected ABAP project.
  • Run Code Search on selected Objects
    The ABAP Code Search plug-in needs to be installed and the ABAP Code Search backend must be available in the selected ABAP project.

If the tree node represents a CDS View, a Database Table or a Database View There are also the following options available via the context menu

  • Open the selected objects in the Data Preview of ADT
  • Open the selected objects in the SAP GUI Transaction DB Browser
    The DB Browser feature needs to be installed in the ABAP System, otherwise the option is not visible

Grouping by Package

You can switch on the grouping by package, which shows the result objects at the correct level in the package hierarchy in which they were created in the system.

Output modes

Some search types (e.g. Method) support a flat output as well. If that is the case you can switch the output mode from Tree to List via the View Menu

Text Filter

For each search type and output mode it is possible to filter the results via a Text Filter.

Special Toolbar Actions

In addition to the default toolbar actions you can reopen the underlying query of the result in the Search Dialog.
There is also the option to save the current query as a favorite (more info on Favorites)
