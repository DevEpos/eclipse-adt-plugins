Where-Used (in CDS) Analysis

The Where-Used Analysis provides a way to display all usages of a CDS View (or Database Table/View) in either the SELECT Part or as Associations in other CDS Views.

There are several options which you can use to control the analysis:

  1. This setting controls whether Uses in SELECT-Parts of a CDS View will be shown
  2. This setting controls whether Uses as Associations of a CDS View will be shown
  3. This toggles a local Tree Filter and shows only the sub trees where a released entity is available
    This feature is only available starting from NW 7.50
  4. Resets a currently active Viewer Filter
  5. Toggles visibility of the Text Filter Control
  6. If active the analysis is searching not only the direct references, but resolve all subsequent references in one step
  7. If this setting is checked only referenced CDS Views which have an API state of type Released will be shown
    This feature is only available starting from NW 7.50
  8. This setting is only relevant if the setting Show Uses in Associations of CDS is active. If checked, the usage is only valid if the association to the triggering CDS View is defined locally (i.e. not propagated)
    This feature is only available starting from NW 7.52

Local Viewer Filter

Additionally you can restrict the result to show only a certain sub tree.

After the filter was applied you see an icon to indicate that the tree is now filtered.

The local filters can be resetted via the action Reset Viewer Filter from the View Menu.

Text Filter

The view also provides the option to filter the results with a Text Filter, which can be activated via the View Menu or the Shortcut Strg+F

Text Filter and Viewer Filter can be combined
