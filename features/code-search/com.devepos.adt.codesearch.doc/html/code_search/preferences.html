Workspace Preferences

General Settings

In the preferences menu you have the possibility to configure the following settings

Search Dialog Settings

  • Settings for Includes
    • Use from previous search
      This option controls whether the control settings for Class Includes, Function Group Includes and Program Includes are taken over from the previously executed search
  • Single Pattern Mode
    • Concatenate lines of the search pattern with '\n' for regular expressions
      This option is relevant if the entered pattern has multiple lines and you have activated the options: Regular Expressions and Single Pattern Mode.
      By activating this option the system will automatically enhance the entered pattern with a new line character after each line of the pattern and consider it a single pattern for the search.
      A recommendation to use this option deactivated is that if you structure very complex patterns you can divide them in different lines for a better overview but you still want it to be considered a single pattern
  • Settings for Transport Request Filter
    The task/transport requests displayed in the autocomplition (or in the F4 value help) can be restricted by default with the following filters:
    • owner set to the logon user
    • status of the task/transport request
    • date of the release

Search Settings

  • Objects to search in a single API call
    For the serial processing of the search, this setting controls how many objects will be searched during a single back end call in an ongoing search query.
